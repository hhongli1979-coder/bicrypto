Extension not showing in admin menu

Symptom: AI Market Maker doesn't appear in the admin navigation after installation.

Solutions:

  • 1. Ensure Ecosystem extension is installed first
  • 2. Run pnpm updator after activation
  • 3. Clear browser cache and refresh
  • 4. Check if extension is enabled in System ‚Üí Extensions

Bots are not executing trades

Symptom: Markets and bots are active but no trades are being executed.

Solutions:

  • 1. Verify cron jobs are running: pm2 list
  • 2. Check global "Enable Trading" setting is ON
  • 3. Verify market status is ACTIVE (not PAUSED or INACTIVE)
  • 4. Ensure bot status is ACTIVE
  • 5. Check if daily trade limit has been reached
  • 6. Review logs for error messages

Market automatically paused

Symptom: Market status changed to PAUSED without manual intervention.

Causes & Solutions:

  • Volatility exceeded: Wait for volatility to normalize or increase threshold
  • Daily loss limit hit: Market will resume at daily reset (midnight UTC)
  • Circuit breaker triggered: Wait for cooldown period
  • Check history logs for the specific reason

No ecosystem markets available to select

Symptom: Dropdown shows no markets when creating an AI market.

Solutions:

  • 1. Create markets in Ecosystem extension first
  • 2. Ensure ecosystem markets are enabled/active
  • 3. Markets already used in AI Market Maker won't appear (one-to-one mapping)

License activation fails

Symptom: Unable to activate the extension with CodeCanyon license.

Solutions:

  • 1. Verify internet connection is stable
  • 2. Double-check CodeCanyon username (case-sensitive)
  • 3. Ensure license code is correct (copy-paste without extra spaces)
  • 4. Check if license is already used on another domain
  • 5. Try disabling any firewall/proxy temporarily

Database or migration errors

Symptom: Errors about missing tables or columns.

Solutions:

  • 1. Run pnpm updator to apply migrations
  • 2. Check database user has ALTER and CREATE permissions
  • 3. Verify database connection in .env file
  • 4. Try running backend build: pnpm build:backend

High CPU or memory usage

Symptom: Server performance degraded after enabling AI Market Maker.

Solutions:

  • 1. Reduce number of active markets
  • 2. Reduce number of bots per market
  • 3. Increase trading interval (e.g., 5 minutes instead of 1)
  • 4. Reduce daily trade limits
  • 5. Enable history cleanup to reduce database size

Trades not appearing in order book

Symptom: AI trades execute but don't show in the trading interface.

Solutions:

  • 1. Check realLiquidityPercent setting - 0% means AI-only (simulated)
  • 2. Increase realLiquidityPercent to create real orders
  • 3. Verify the ecosystem market has trading enabled
  • 4. Check frontend WebSocket connection is active

üìã Analyzing Logs

Check system logs for detailed error information:

View PM2 logs:
pm2 logs backend --lines 100
Filter for AI Market Maker errors:
pm2 logs backend --lines 500 | grep -i "ai.*trading\|market.*maker"
View trade history in admin:
/admin/ai/market-maker/history

API Integration Guide

Complete Developer Documentation for Payment Gateway API

API Overview

The Payment Gateway API allows you to accept cryptocurrency payments on your website or application. The API follows REST conventions and uses JSON for request and response bodies.

Base URL: https://your-platform.com/api/gateway/v1

Authentication

All API requests must include your API keys in the request headers: 

X-Api-Public-Key: pk_live_your_public_key
X-Api-Secret-Key: sk_live_your_secret_key

Live Mode Keys

Prefix: pk_live_ / sk_live_
Use for production payments

Test Mode Keys

Prefix: pk_test_ / sk_test_
Use for development/testing

Security Warning: Never expose your Secret Key in client-side code. Always make API calls from your server.

Create Payment Intent

POST /payment-intents

Creates a new payment intent that represents a customer's intent to pay. Returns a checkout URL where the customer completes payment.

Request Body

{
  "amount": 99.99,
  "currency": "USD",
  "orderId": "ORDER-12345",
  "description": "Premium subscription",
  "customerEmail": "customer@example.com",
  "customerName": "John Doe",
  "successUrl": "https://yoursite.com/success",
  "cancelUrl": "https://yoursite.com/cancel",
  "metadata": {
    "productId": "prod_123",
    "customField": "value"
  }
}

Parameters

Parameter Type Required Description
amount number Yes Payment amount in the specified currency
currency string Yes Three-letter ISO currency code (e.g., USD, EUR)
orderId string No Your internal order identifier
description string No Description shown to customer
customerEmail string No Customer's email address
customerName string No Customer's full name
successUrl string Yes URL to redirect after successful payment
cancelUrl string Yes URL to redirect if payment is cancelled
metadata object No Custom key-value pairs (returned in webhooks)

Response

{
  "success": true,
  "data": {
    "paymentIntentId": "pi_abc123xyz",
    "checkoutUrl": "https://platform.com/gateway/checkout/pi_abc123xyz",
    "amount": 99.99,
    "currency": "USD",
    "status": "PENDING",
    "expiresAt": "2024-12-07T12:00:00Z"
  }
}

Retrieves the current status and details of a payment intent.

Payment Status Values

PENDING

Awaiting customer payment

COMPLETED

Payment successful

Payment failed

Cancelled by customer

REFUNDED

Fully refunded

Create Refund

POST /refund

Creates a full or partial refund for a completed payment.

Request Body

{
  "paymentIntentId": "pi_abc123xyz",
  "amount": 50.00,
  "reason": "Customer requested refund"
}

Parameters

Parameter Type Required Description
paymentIntentId string Yes ID of the payment to refund
amount number No Amount to refund (full refund if omitted)
reason string No Reason for the refund

Response

{
  "success": true,
  "data": {
    "refundId": "ref_xyz789",
    "paymentIntentId": "pi_abc123xyz",
    "amount": 50.00,
    "status": "COMPLETED",
    "reason": "Customer requested refund"
  }
}

Webhooks

Webhooks notify your server of payment events in real-time. Configure your webhook URL in the merchant dashboard.

Webhook Events

  • payment.completed - Payment was successful
  • payment.failed - Payment failed
  • payment.cancelled - Customer cancelled
  • payment.expired - Session expired
  • refund.completed - Refund processed

Webhook Payload

{
  "event": "payment.completed",
  "timestamp": "2024-12-06T10:30:00Z",
  "data": {
    "paymentIntentId": "pi_abc123xyz",
    "orderId": "ORDER-12345",
    "amount": 99.99,
    "currency": "USD",
    "status": "COMPLETED",
    "customerEmail": "customer@example.com",
    "feeAmount": 2.50,
    "netAmount": 97.49,
    "metadata": {
      "productId": "prod_123"
    }
  }
}

Webhook Signature Verification

Verify webhook authenticity using the signature header: 

X-Webhook-Signature: sha256=abc123...

Verification in PHP: 

$payload = file_get_contents('php://input');
$signature = $_SERVER['HTTP_X_WEBHOOK_SIGNATURE'];
$expected = 'sha256=' . hash_hmac('sha256', $payload, $webhookSecret);

if (hash_equals($expected, $signature)) {
    // Signature valid, process webhook
    $data = json_decode($payload, true);
    // Update order status based on $data['event']
} else {
    // Invalid signature, reject
    http_response_code(401);
}

Responding to Webhooks

Your endpoint should return a 200 status code within 30 seconds. Failed webhooks are retried up to 3 times with exponential backoff.

Code Examples

PHP Example

<?php
// Create a payment intent
$apiUrl = 'https://your-platform.com/api/gateway/v1/payment-intents';

$data = [
    'amount' => 99.99,
    'currency' => 'USD',
    'orderId' => 'ORDER-' . time(),
    'description' => 'Premium subscription',
    'customerEmail' => 'customer@example.com',
    'successUrl' => 'https://yoursite.com/success',
    'cancelUrl' => 'https://yoursite.com/cancel'
];

$ch = curl_init($apiUrl);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data));
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'Content-Type: application/json',
    'X-Api-Public-Key: pk_live_your_public_key',
    'X-Api-Secret-Key: sk_live_your_secret_key'
]);

$response = curl_exec($ch);
curl_close($ch);

$result = json_decode($response, true);

if ($result['success']) {
    // Redirect customer to checkout
    header('Location: ' . $result['data']['checkoutUrl']);
    exit;
} else {
    // Handle error
    echo 'Error: ' . $result['message'];
}

Node.js Example

const axios = require('axios');

async function createPayment() {
    try {
        const response = await axios.post(
            'https://your-platform.com/api/gateway/v1/payment-intents',
            {
                amount: 99.99,
                currency: 'USD',
                orderId: `ORDER-${Date.now()}`,
                description: 'Premium subscription',
                customerEmail: 'customer@example.com',
                successUrl: 'https://yoursite.com/success',
                cancelUrl: 'https://yoursite.com/cancel'
            },
            {
                headers: {
                    'Content-Type': 'application/json',
                    'X-Api-Public-Key': 'pk_live_your_public_key',
                    'X-Api-Secret-Key': 'sk_live_your_secret_key'
                }
            }
        );

        if (response.data.success) {
            // Redirect customer to checkout URL
            return response.data.data.checkoutUrl;
        }
    } catch (error) {
        console.error('Payment error:', error.response?.data || error.message);
    }
}

Python Example

import requests
import time

def create_payment():
    url = 'https://your-platform.com/api/gateway/v1/payment-intents'

    headers = {
        'Content-Type': 'application/json',
        'X-Api-Public-Key': 'pk_live_your_public_key',
        'X-Api-Secret-Key': 'sk_live_your_secret_key'
    }

    data = {
        'amount': 99.99,
        'currency': 'USD',
        'orderId': f'ORDER-{int(time.time())}',
        'description': 'Premium subscription',
        'customerEmail': 'customer@example.com',
        'successUrl': 'https://yoursite.com/success',
        'cancelUrl': 'https://yoursite.com/cancel'
    }

    response = requests.post(url, json=data, headers=headers)
    result = response.json()

    if result['success']:
        return result['data']['checkoutUrl']
    else:
        raise Exception(result.get('message', 'Payment creation failed'))

Error Handling

API errors return a consistent JSON structure: 

{
  "success": false,
  "message": "Invalid API key",
  "code": "INVALID_API_KEY"
}

Common Error Codes

HTTP Code Error Code Description
400 VALIDATION_ERROR Invalid request parameters
401 INVALID_API_KEY Missing or invalid API keys
403 MERCHANT_SUSPENDED Merchant account is suspended
404 NOT_FOUND Resource not found
409 ALREADY_REFUNDED Payment already fully refunded
500 INTERNAL_ERROR Server error, try again later

Rate Limits

API requests are rate limited to ensure service stability:

  • Standard: 100 requests per minute per API key
  • Payment creation: 30 requests per minute
  • Refunds: 10 requests per minute

Rate limit headers are included in all responses: 

X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1701858000

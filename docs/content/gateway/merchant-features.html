Merchant Features

Complete Guide for Payment Gateway Merchants

Merchant Dashboard Overview

The merchant dashboard provides a complete interface for managing your payment gateway integration. Access it by navigating to Gateway from the main navigation after logging into your account.

From the dashboard, merchants can view real-time analytics, manage API keys, track payments, request payouts, and configure their integration settings.

Getting Started as a Merchant

1

Create Your Merchant Account

When you first access the Gateway section, you'll be prompted to create a merchant account:

  • Business Name - Your company or store name (displayed to customers)
  • Business Email - Contact email for payment notifications
  • Website URL - Your website address for verification
  • Description - Brief description of your business (optional)
2

Generate API Keys

Navigate to Gateway > API Keys to create your integration credentials:

  • Public Key - Used for client-side operations (safe to expose)
  • Secret Key - Used for server-side API calls (keep secure!)
  • Webhook Secret - Used to verify webhook signatures

Security Note: Never expose your Secret Key in client-side code. Always make API calls from your server.

3

Configure Webhook URL

Set up your webhook endpoint to receive real-time payment notifications: 

https://yourwebsite.com/webhook/bicrypto-gateway

Your webhook endpoint will receive POST requests with payment status updates.

Dashboard Analytics

Overview Metrics

  • Total Revenue - Cumulative payment volume
  • Total Payments - Number of completed transactions
  • Average Order Value - Mean transaction amount
  • Success Rate - Percentage of successful payments

Balance Information

  • Available Balance - Ready for payout
  • Pending Balance - Awaiting settlement
  • Total Fees - Platform fees deducted
  • Total Payouts - Amount already withdrawn

Payment Management

Viewing Payments

Navigate to Gateway > Payments to view all your transactions. The payments list shows:

  • Payment ID - Unique identifier for tracking
  • Order ID - Your external order reference
  • Amount - Transaction amount and currency
  • Status - Current payment state (Pending, Completed, Failed, Refunded)
  • Customer - Customer email address
  • Date - Transaction timestamp

Payment Details

Click on any payment to view detailed information:

  • Payment Wallets - Which wallet(s) the customer used
  • Fee Breakdown - Platform fee and net amount
  • Conversion Details - Original currency and conversion rate
  • Timeline - Status history with timestamps

Issuing Refunds

Merchants can issue full or partial refunds for completed payments:

  1. Open the payment details
  2. Click "Refund Payment"
  3. Enter the refund amount (full or partial)
  4. Provide a reason for the refund
  5. Confirm the refund

Note: Refunds return funds to the customer's original wallet(s) using the stored allocation data. Platform fees are also refunded proportionally.

Live Mode

  • Deducts from customer wallets
  • Updates merchant balances
  • Sends webhook notifications
  • Creates transaction records

Test Mode

  • Simulates payment flow
  • No real fund transfers
  • Separate test payment history
  • Still sends webhooks (for testing)
  • Safe for development

Use the mode toggle in the top navigation bar to switch between Live and Test modes. Each mode has separate API keys - use the appropriate keys in your integration based on the environment.

Payout Management

Understanding Payouts

When customers complete payments, the net amount (after platform fees) is credited to your gateway pending balance. This represents funds that are awaiting payout and is tracked separately from your regular wallet balance.

Automatic Payouts

Depending on the platform configuration, payouts may be processed automatically on a schedule (daily, weekly, or monthly). When a payout is processed:

  1. Your gateway pending balance is checked against the minimum payout threshold
  2. A payout request is created automatically
  3. Admin approves the payout (if required)
  4. Funds are transferred to your available wallet balance

Payout Status

PENDING

PROCESSING

COMPLETED

# FAQ v6.0.1

**Release Date:** December 2025
**Tags:** DEDICATED PAGES, WYSIWYG EDITOR, UX IMPROVEMENTS

## ⚠️ Important: Core Compatibility

This release requires **Bicrypto Core v6.0.1 or higher**. Ensure your core platform is updated before installing this FAQ extension update.

## Overview

Version 6.0.1 improves the FAQ management workflow by converting drawer-based editing to dedicated full-page experiences and integrating the new WYSIWYG editor from Core v6.0.1.

## Major Changes

### Dedicated Question Answer Page

- **Full-Page Answer Editor**: Question answers now open in a dedicated page at `/admin/faq/question/[id]/answer` instead of a drawer
- **Better Focus**: Full-screen layout provides more space for writing quality answers
- **Removed Answer Dialog Component**: Eliminated cramped drawer-based answer editor

### Dedicated FAQ Conversion Page

- **Full-Page Conversion Flow**: Converting user questions to FAQs now uses a dedicated page at `/admin/faq/question/[id]/convert`
- **Enhanced Review Process**: More space to review and edit questions before converting to FAQ
- **Removed Convert Dialog Component**: Eliminated limited drawer-based conversion flow

### WYSIWYG Editor Integration

- **Professional Content Editor**: FAQ answers now use the new WYSIWYG editor from Core v6.0.1
- **Rich Text Formatting**: Support for headings, bold, italic, lists, links, images, code blocks, and tables
- **Media Library Access**: Direct access to media library for inserting images into FAQ answers
- **Consistent Experience**: Same editor used across FAQs, blog posts, and landing pages

## Breaking Changes

### Removed Components

- **Deleted `answer-question-dialog.tsx`**: Answer dialog removed in favor of dedicated page
- **Deleted `convert-to-faq-dialog.tsx`**: Convert dialog removed in favor of dedicated page

### Updated Workflows

- **New Navigation**: Admins navigate to dedicated pages instead of opening drawers
- **URL Structure Changes**:
  - Answer page: `/admin/faq/question/[id]/answer`
  - Convert page: `/admin/faq/question/[id]/convert`
